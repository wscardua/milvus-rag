# Checklist — Django Web (RAG)

- a tela de upload valida tipo, tamanho e metadados obrigatórios?
- a ação de upload/consulta usa os contratos da API FastAPI?
- a UI evita fazer chunking/embeddings/retrieval localmente?
- o estado de ingestão do documento é exibido de forma clara?
- as respostas do RAG mostram as citações (documento/trecho de origem)?
- o admin segue as mesmas validações do domínio?
- mudança visual que altera comportamento voltou para a spec?
